# AGT-05 - Textual TUI — Input Panel, Answer Panel, Agent Trace Panel

- Epic: Epic 2 - Agentic System (Act 1)

## Objective

Create the Textual-based terminal UI that drives the demo, showing the user's question, the agent's answer, and a live trace of which sub-agents and tools were called.

## Description

The TUI is the audience-facing surface for all three acts of the demo. It must suppress stdout (loguru writes to file only) so Textual owns the terminal cleanly. The three-panel layout — input, answer, trace — lets the presenter type a question, see the structured answer, and show the agent execution path in real time. The trace panel is critical for Act 2 (showing wrong SQL) and Act 3 (showing corrected SQL after sync).

## Scope

- Create `agentic_system/tui/app.py` with the Textual `App` subclass and the three-panel layout.
- Create `agentic_system/tui/widgets.py` with any custom Textual widgets needed for the panels.
- Input panel: single-line text input that submits on Enter.
- Answer panel: scrollable display for the orchestrator's synthesised answer.
- Trace panel: scrollable display showing tool call order, sub-agent names, SQL generated, and row counts.
- Wire the TUI to the orchestrator — submitting a question triggers `Runner.run(orchestrator, input)`.
- Suppress all stdout output; Textual must own the terminal (loguru file-only enforced by AGT-06).

## Out Of Scope

- MLflow panel display (MLflow UI is a separate browser window, not embedded in TUI).
- loguru configuration itself (AGT-06).
- MLflow tracer decorator (AGT-07).
- `agentic_system/main.py` entry point that launches the app (AGT-08).
- Gradio web UI variant (noted as nice-to-have in backlog, out of core scope).

## Deliverables

- `agentic_system/tui/app.py` — Textual `App` with three-panel layout wired to the orchestrator.
- `agentic_system/tui/widgets.py` — custom widgets for answer and trace panels if needed.

## Acceptance Criteria

- TUI launches without errors when `python agentic_system/main.py` is run.
- Typing a question and pressing Enter submits it to the orchestrator asynchronously.
- Answer panel updates with the orchestrator's response after the run completes.
- Trace panel shows at minimum: which tools were called, in which order, and the SQL string generated by AgentNL2SQL.
- No agent output appears on stdout — all terminal output is rendered by Textual widgets.

## Dependencies

- AGT-03: Root orchestrator importable and callable.
- AGT-06: loguru configured to suppress stdout before TUI launches.
- AGT-01, AGT-02: Sub-agents functional so trace data is available.

## Assumptions

- Textual is the chosen TUI framework (already specified in backlog architecture).
- Orchestrator calls are run via the OpenAI Agents SDK `Runner` in an async context compatible with Textual's event loop.

## Notes

- The trace panel content is central to the demo narrative — ensure tool call order and the SQL string are clearly visible.
- Keep widget code minimal; custom widgets only where Textual built-ins are insufficient.
