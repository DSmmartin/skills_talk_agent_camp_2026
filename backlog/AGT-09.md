# AGT-09 - End-to-End — Ghost Contributor Query Returns Correct Rows

## Objective

Confirm that the complete Act 1 demo path works: the orchestrator answers "Who's churning contributors on kubernetes/kubernetes?" with correct SQL using `merged = 1` and returns non-empty rows.

## Description

This is the integration acceptance task for Epic 2. All components — ClickHouse, ChromaDB, the orchestrator, AgentRAG, AgentNL2SQL, TUI, loguru, and MLflow — must work together for the Act 1 demo narrative to land. The success state is the TUI showing a populated answer panel with real row data, the trace panel showing the two tool calls in order, and the MLflow UI showing a completed run. This task also covers the automated test counterpart in `test_use_case.py` under the `pre_migration` mark.

## Scope

- Manual smoke test: run `make up && make seed && make seed-vectors && python agentic_system/main.py`, submit the ghost contributor question, and verify correct rows are returned.
- Verify the TUI trace panel shows `retrieve_schema_context` called before `query_github_data`.
- Verify the SQL generated by AgentNL2SQL contains `merged = 1` (pre-migration correct predicate).
- Verify MLflow UI at `http://localhost:5000` shows the run with nested spans for all three agents.
- Ensure `pytest -m pre_migration` passes against the mock fixtures in `tests/test_use_case.py`.

## Out Of Scope

- Act 2 broken-state verification (MIG-05 in Epic 3).
- Act 3 schema-sync verification (DEV-14 in Epic 4).
- Performance benchmarking or load testing.
- Cross-platform cold-start validation (INF-09, INF-10 already in Epic 1).

## Deliverables

- Passing `pytest -m pre_migration` test suite (`tests/test_use_case.py`).
- Confirmed manual smoke test: TUI returns non-empty ghost contributor rows for the kubernetes/kubernetes question.
- Any bug fixes discovered during integration across AGT-01 through AGT-08.

## Acceptance Criteria

- `pytest -m pre_migration` passes with no failures.
- `test_ghost_contributor_query_pre_migration`: SQL contains `merged = 1` and result rows are non-empty.
- `test_rag_returns_correct_field_pre_migration`: RAG context contains `merged` and `UInt8`, does not contain `merged_at`.
- TUI manual test: answer panel shows repository names and actor logins; trace panel shows tool call sequence.
- MLflow UI shows a run with `orchestrator`, `AgentRAG`, and `AgentNL2SQL` spans, each with token and latency attributes.

## Dependencies

- AGT-01 through AGT-08: All Epic 2 components complete.
- INF-01 through INF-08: Infrastructure fully running (ClickHouse, ChromaDB, MLflow seeded).
- `tests/conftest.py` with mock fixtures for ClickHouse, ChromaDB, and OpenAI (test infrastructure).

- The GitHub Archive sample slice loaded by `make seed` contains PR events for `kubernetes/kubernetes`.
- Mock fixtures in `tests/` mirror the live pre-migration schema state accurately enough for CI.
